Design system con variabili CSS e shadcn

Ritchie 5 Febbraio 2026 · 4 min di lettura

Le CSS custom properties (variabili CSS) sono lo strumento perfetto per costruire un design system leggero e flessibile. Ispirandoci a shadcn/ui, possiamo replicare l'intero sistema di colori usando valori HSL.

Perché HSL?

Il formato HSL (Hue, Saturation, Lightness) permette di manipolare i colori in modo intuitivo. shadcn/ui memorizza i valori senza la funzione hsl() nelle variabili, così possiamo aggiungere l'opacità quando serve:

:root {
  --primary: 240 5.9% 10%;
}

.element {
  background: hsl(var(--primary));
  border: 1px solid hsl(var(--primary) / 0.2);
}

Struttura delle variabili

Il sistema si basa su coppie semantiche: ogni colore ha il suo -foreground per garantire il contrasto.

  • --background / --foreground — colori base della pagina
  • --primary / --primary-foreground — azioni principali
  • --muted / --muted-foreground — elementi secondari
  • --border / --input / --ring — bordi e focus

Dark mode

Il bello di questo approccio: per il tema scuro basta ridefinire le stesse variabili sotto la classe .dark. Tutto il CSS dei componenti resta invariato.

Un buon design system non è quello con più componenti, ma quello che risolve il 90% dei casi con il minimo codice.

Esempio pratico

Un bottone che funziona in entrambi i temi senza una riga di CSS in più:

.btn-default {
  background-color: hsl(var(--primary));
  color: hsl(var(--primary-foreground));
}

In tema chiaro sarà scuro su bianco, in tema scuro sarà bianco su scuro. Zero media queries, zero duplicazioni.
